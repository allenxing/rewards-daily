# next-intl 国际化方案

## 1. Goal

将所有用户可见文案提取到 next-intl JSON 语言文件，cookie 切换中/英双语，未来可横向扩展。

## 2. Stack

- `next-intl` (latest, App Router)
- Locale 检测: cookie `NEXT_LOCALE` (默认 `zh`, 切换后 `en`)
- PPR 兼容: 不走 middleware, 不走 URL 路径前缀, cookie 读取在 `i18n/request.ts`

## 3. 文件结构

```
messages/
  zh.json        ← 中文基准 (完整覆盖所有文案)
  en.json        ← 英文 (第一期覆盖核心,逐步补齐)
lib/i18n.ts      ← next-intl getRequestConfig
components/common/locale-toggle.tsx  ← 语言切换组件
```

### 3.1 `lib/i18n.ts`

```ts
import { getRequestConfig } from "next-intl/server";
import { cookies } from "next/headers";

export default getRequestConfig(async () => {
  const locale = (await cookies()).get("NEXT_LOCALE")?.value ?? "zh";
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

### 3.2 `next.config.ts` 变更

```ts
import createNextIntlPlugin from "next-intl/plugin";
const withNextIntl = createNextIntlPlugin("./lib/i18n.ts");
// ... 保持 cacheComponents: true 等现有配置
export default withNextIntl({ cacheComponents: true, ... });
```

### 3.3 Root layout 变更

```tsx
// app/layout.tsx
import { NextIntlClientProvider } from "next-intl";
import { getMessages } from "next-intl/server";

// 在现有 ThemeProvider 外层加 NextIntlClientProvider
```

### 3.4 Locale Toggle

- cookie key: `NEXT_LOCALE`
- `locale-toggle.tsx`: 两个按钮 (中文 / English), 点击设 cookie + reload
- 放置在 landing 页 + admin sidebar

### 命名约定

- 命名空间按 `{端}.{页面}.{组件/场景}`
- toast 消息统一在各自页面命名空间下的 `toast` 子对象
- 全局通用词放 `common`
- server action 错误消息统一放 `error`

## 5. 文案使用模式

### Server Component

```tsx
import { getTranslations } from "next-intl/server";

export default async function AdminTasksPage() {
  const t = await getTranslations("admin.tasks");
  return <h1>{t("pageTitle")}</h1>;
}
```

### Client Component

```tsx
"use client";
import { useTranslations } from "next-intl";

export function TasksClient({ ... }) {
  const t = useTranslations("admin.tasks");
  return <button>{t("add")}</button>;
}
```

### 带参数

```json
{ "greeting": "嗨, {name}!" }
```
```ts
t("greeting", { name: child.name });
```

### Server Action 错误

Server actions 返回 error code 而不是翻译后的文字, 由 consumer 层用 t() 翻译:

```ts
// lib/actions.ts
return { ok: false, code: "INSUFFICIENT_POINTS" };

// 调用方
const r = await adjustPointsAction(fd);
if (!r.ok) toast.error(t(r.code));
```

在 `zh.json` 中用 error code 做 key:
```json
{
  "error": {
    "INSUFFICIENT_POINTS": "积分不足",
    "WISH_NOT_FOUND": "愿望不存在",
    ...
  }
}
```

## 6. 迁移策略

### Step 1: 基础设施
| 文件 | 操作 |
|------|------|
| `package.json` | `npm install next-intl` |
| `lib/i18n.ts` | 新建 |
| `next.config.ts` | 加 `withNextIntl` 包装 |
| `app/layout.tsx` | 加 `NextIntlClientProvider` |
| `messages/zh.json` | 新建,含全部文案 |
| `messages/en.json` | 新建,先写中文占位 |
| `components/common/locale-toggle.tsx` | 新建 |

### Step 2: 文案替换 (~35 文件)
按页面优先级:
1. Admin 端 (app/admin/*, components/admin/*) — ~15 文件
2. Child 端 (app/child/*, components/child/*) — ~8 文件
3. Landing 页 (components/landing/*) — ~6 文件
4. Server actions + errors (lib/actions.ts, lib/utils/errors.ts) — ~2 文件
5. 剩余的 auth 页、toast 等 — ~4 文件

### Step 3: en.json 翻译
将 en.json 中的中文占位替换为英文。

## 7. PPR/cacheComponents 注意事项

- `next-intl` 的 `getRequestConfig` 默认只在 `i18n/request.ts` 中调用,不走 middleware
- `<NextIntlClientProvider>` 在 root layout 中用 `getMessages()` 读取一次,不会在每个 page 重新调用
- 动态 locale 切换靠页面刷新 (reload),不会在 PPR 流中动态改变,所以不会造成 PPR 缓存不一致
- Cookie 读取在 `i18n/request.ts` 而非 middleware,不影响 `proxy.ts`
